Source code for our paper **"Integrating Structural and Temporal Information of Trace Cases: A Multi-Perspective Framework for Process Representation Learning"**. We propose a multi-perspective process representation learning framework (**STAMP**) that integrates structural and temporal information of traces via two self-supervised tasks. It models control flows as directed subgraphs for structural dependencies, and converts multi-dimensional event attributes into 2D images via Gramian Angular Fields (GAF) to extract dynamic features using a CNN. Both embeddings are fused into a discriminative trace representation to support downstream trace clustering and process discovery.

## Project Structure

```
.
├── mppn/                       # Core Python package (generated by nbdev, mppn baseline package)
│   ├── preprocessing.py        # Event log import, cleaning, splitting, feature processing, DataLoader construction
│   ├── pipeline.py             # Training/validation workflow, PPModel base class, runner for batch evaluation
│   ├── mppn.py                 # MPPN model definition and PPM_MPPN adapter
│   ├── baselines.py            # Multiple PPM baseline models and attr_dict
│   ├── graph2vec.py            # Graph embedding implementation
│   ├── clustering.py           # Clustering and evaluation metrics (ARI/NMI/B3, etc.)
│   ├── eventlog.py             # XES/JSON log loading and preprocessing
│   └── b3.py                   # B3 metric computation
├── *.ipynb                     # Research/experiment notebooks (traceEmbedding/clustering/test_mppn, etc.)
├── datasets/                   # Datasets (Due to size and licensing, not included in repo, but instructions for downloading are provided)
└── README.md                   # Basic installation instructions
```

## Usage

### 1 Environment Setup and Installation

It is recommended to use Python 3.7

```bash
conda create -n traceEmbedding python=3.7
conda activate traceEmbedding
python -m pip install -U pip
pip install -r requirements.txt
```
Notes :Dependency versions are defined in the `requirements.txt`.

---

### 2 Running Jupyter Notebooks

Recommended notebooks:

* `traceEmbedding.ipynb`: Main pipeline and embedding experiments
* `EventLog.ipynb`: Event log processing and graph generation
* `clustering.ipynb`: Clustering and evaluation
* `LLM-embedding-clustering.ipynb`: LLM Baseline Pipeline
* `test_mppn.ipynb`: Baseline Model and evaluation workflow examples

## Datasets

This project uses two public event log datasets:

---

### BPIC 2015 (Municipal Building Permit Process)

* **Description**: Event logs from building permit processes across five Dutch municipalities, containing 5,649 traces
* **Clustering labels**: Case-level attribute `cluster:label` (5 categories)
* **Event attributes**:
  * `concept:name` (activity)
  * `org:resource` (resource)
  * `monitoringResource`
* **Note**: Download from https://data.4tu.nl/collections/BPI\_Challenge\_2015/5065424
---

### BPIC 2019 (Procure-to-Pay Process)

* **Description**: Procure-to-pay (P2P) process of a multinational company, with cases distinguished by attributes such as `Item Type`, `Document Type`, and `Item Category`
* **Clustering labels**: `Item Category`
* **Event attributes**:
  * `concept:name`
  * `org:resource`
* **Note**: Download from https://data.4tu.nl/articles/dataset/BPI\_Challenge\_2019/12715853
